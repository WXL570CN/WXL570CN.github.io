useMutation

# useMutation

const {  data,  error,  isError,  isIdle,  isLoading,  isPaused,  isSuccess,  failureCount,  failureReason,  mutate,  mutateAsync,  reset,  status,} = useMutation({  mutationFn,  cacheTime,  mutationKey,  networkMode,  onError,  onMutate,  onSettled,  onSuccess,  retry,  retryDelay,  useErrorBoundary,  meta})
mutate(variables, {  onError,  onSettled,  onSuccess,})

选项

  • mutationFn: (变量: TVariables) => Promise<TData>
    • 必需，但前提是未定义默认突变函数
    • 执行异步任务并返回承诺的函数。
    • variables 是 mutate 将传递给您的 mutationFn 的对象
  • 缓存时间：数字 |无穷
    • 未使用/非活动缓存数据保留在内存中的时间（以毫秒为单位）。当突变的缓存变为未使用或不活动时，该缓存数据将在这段时间后被垃圾收集。当指定不同的缓存时间时，将使用最长的一个。
    • 如果设置为 Infinity，将禁用垃圾收集
  • 突变键：字符串
    • 选修的
    • 可以将突变键设置为继承使用 queryClient.setMutationDefaults 设置的默认值或在开发工具中识别突变。
  • 网络模式：'在线' | '总是' | '离线优先
    • 选修的
    • 默认为“在线”
    • 有关详细信息，请参阅网络模式。
  • onMutate: (变量: TVariables) => Promise<TContext |无效> |上下文 |空白
    • 选修的
    • 此函数将在触发变异函数之前触发，并传递变异函数将接收的相同变量
    • 用于对资源执行乐观更新以希望突变成功
    • 如果发生突变失败，此函数返回的值将传递给 onError 和 onSettled 函数，并且可用于回滚乐观更新。
  • onSuccess：（数据：TData，变量：TVariables，上下文？：TContext）=> Promise<unknown> |未知
    • 选修的
    • 该函数将在突变成功时触发，并将传递突变的结果。
    • 如果返回一个承诺，它将在继续之前等待并解决
  • onError: (err: TError, variables: TVariables, context?: TContext) => Promise<未知> |未知
    • 选修的
    • 如果突变遇到错误并将传递错误，则此函数将触发。
    • 如果返回一个承诺，它将在继续之前等待并解决
  • onSettled: (data: TData, error: TError, variables: TVariables, context?: TContext) => Promise<未知> |未知
    • 选修的
    • 当成功获取突变或遇到错误并传递数据或错误时，此函数将触发
    • 如果返回一个承诺，它将在继续之前等待并解决
  • 重试：布尔值 |编号 | (failureCount: number, error: TError) => 布尔值
    • 默认为 0。
    • 如果为假，失败的突变将不会重试。
    • 如果为真，失败的突变将无限重试。
    • 如果设置为数字，例如3，失败的突变会重试，直到失败的突变计数满足该数量。
  • 重试延迟：数字 | (retryAttempt: number, error: TError) => number
    • 此函数接收一个 retryAttempt 整数和实际错误，并返回在下一次尝试之前应用的延迟（以毫秒为单位）。
    • 像 attempt => Math.min(attempt > 1 ? 2 ** attempt * 1000 : 1000, 30 * 1000) 这样的函数应用指数退避。
    • 像 attempt => attempt * 1000 这样的函数应用线性退避。
  • 使用错误边界：未定义 |布尔 | （错误：TError）=> 布尔值
    • 默认为全局查询配置的 useErrorBoundary 值，未定义
    • 如果您希望在渲染阶段抛出变异错误并传播到最近的错误边界，请将此设置为 true
    • 将此设置为 false 以禁用将错误抛出到错误边界的行为。
    • 如果设置为函数，它将传递错误并返回一个布尔值，指示是在错误边界中显示错误 (true) 还是将错误作为状态返回 (false)
  • 元：记录<字符串，未知>
    • 选修的
    • 如果设置，则存储有关可根据需要使用的突变缓存条目的附加信息。只要突变可用，它就可以访问（例如 MutationCache 的 onError、onSuccess 函数）。
  • 上下文？：React.Context<QueryClient |未定义>
    • 使用它来使用自定义 React 查询上下文。否则，将使用 defaultContext。

Returns

  • 变异：（变量：TVariables，{onSuccess，onSettled，onError}）=>无效
    • 您可以使用变量调用突变函数来触发突变，并可选择挂钩其他回调选项。
    • 变量：TV变量
      • 选修的
      • 要传递给 mutationFn 的变量对象。
    • onSuccess:（数据：TData，变量：TVariables，上下文：TContext）=>无效
      • 选修的
      • 该函数将在突变成功时触发，并将传递突变的结果。
      • void 函数，返回值将被忽略
    • onError: (err: TError, variables: TVariables, context: TContext | undefined) => 无效
      • 选修的
      • 如果突变遇到错误并将传递错误，则此函数将触发。
      • void 函数，返回值将被忽略
    • onSettled：（数据：TData | undefined，错误：TError | null，变量：TVariables，上下文：TContext | undefined）=> void
      • 选修的
      • 当成功获取突变或遇到错误并传递数据或错误时，此函数将触发
      • void 函数，返回值将被忽略
    • 如果您发出多个请求，onSuccess 只会在您最近一次调用后触发。
  • mutateAsync: (变量: TVariables, { onSuccess, onSettled, onError }) => Promise<TData>
    • 类似于 mutate 但返回一个可以等待的承诺。
  • 状态：字符串
    • 将：
      • 变异函数执行前的空闲初始状态。
      • 如果当前正在执行突变，则加载。
      • 如果最后一次突变尝试导致错误，则为错误。
      • 如果最后一次突变尝试成功，则成功。
  • isIdle、isLoading、isSuccess、isError：从状态派生的布尔变量
  • 暂停：布尔值
    • 如果突变已暂停，则为真
    • 有关详细信息，请参阅网络模式。
  • 数据：未定义 |未知
    • 默认为未定义
    • 查询的最后一次成功解析的数据。
  • 错误：空 |恐怖
    • 如果遇到错误，查询的错误对象。
  • 重置：（）=>无效
    • 清除突变内部状态的函数（即，它将突变重置为其初始状态）。
  • 失败计数：数字
    • 突变的失败计数。
    • 每次突变失败时增加。
    • 突变成功时重置为 0。
  • 失败原因：空 |恐怖
    • 突变重试的失败原因。
    • 突变成功时重置为 null。
