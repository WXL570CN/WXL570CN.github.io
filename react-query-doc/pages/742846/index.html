# MutationCache

MutationCache 是突变的存储。

通常，您不会直接与 MutationCache 交互，而是使用 QueryClient。

import { MutationCache } from '@tanstack/react-query'
const mutationCache = new MutationCache({  onError: error => {    console.log(error)  },  onSuccess: data => {    console.log(data)  },})

它可用的方法有：

  • 得到所有 (opens new window)
  • 订阅 (opens new window)
  • 清除 (opens new window)

选项

  • onError?: (error: unknown, variables: unknown, context: unknown, mutation: Mutation) => Promise<未知> |未知
    • 选修的
    • 如果某些突变遇到错误，将调用此函数。
    • 如果您从中返回一个 Promise，它将被等待
  • onSuccess?: (data: unknown, variables: unknown, context: unknown, mutation: Mutation) => Promise<未知> |未知
    • 选修的
    • 如果某些突变成功，将调用此函数。
    • 如果您从中返回一个 Promise，它将被等待
  • onSettled?: (data: unknown | undefined, error: unknown | null, variables: unknown, context: unknown, mutation: Mutation) => 承诺<未知> |未知
    • 选修的
    • 如果某些突变已解决（成功或错误），将调用此函数。
    • 如果您从中返回一个 Promise，它将被等待
  • onMutate?: (variables: unknown, mutation: Mutation) => Promise<unknown> |未知
    • 选修的
    • 这个函数将在一些突变执行之前被调用。
    • 如果您从中返回一个 Promise，它将被等待

[

# 全局回调

](https://tanstack.com/query/latest/docs/react/reference/MutationCache#global-callbacks)

MutationCache 上的 onError、onSuccess、onSettled 和 onMutate 回调可用于在全局级别处理这些事件。它们与提供给 QueryClient 的 defaultOptions 不同，因为：

  • defaultOptions 可以被每个 Mutation 覆盖——全局回调将始终被调用。
  • onMutate 不允许返回上下文值。

[

# 突变缓存.getAll

](https://tanstack.com/query/latest/docs/react/reference/MutationCache#mutationcachegetall)

getAll 返回缓存中的所有突变。

注意：大多数应用程序通常不需要这，但在极少数情况下需要有关突变的更多信息时可以派上用场

const mutations = mutationCache.getAll()

Returns

  • 突变[]
    • 来自缓存的变异实例

[

# mutationCache.subscribe 订阅

](https://tanstack.com/query/latest/docs/react/reference/MutationCache#mutationcachesubscribe)

subscribe 方法可用于订阅整个突变缓存，并获知对缓存的安全/已知更新，例如突变状态更改或突变被更新、添加或删除。

const callback = event => {  console.log(event.type, event.mutation)}
const unsubscribe = mutationCache.subscribe(callback)

选项

  • 回调：（突变？：MutationCacheNotifyEvent）=>无效
    • 每当更新时，都会使用突变缓存调用此函数。

Returns

  • 退订：功能=>无效
    • 此函数将从突变缓存中取消订阅回调。

[

# 突变缓存.clear

](https://tanstack.com/query/latest/docs/react/reference/MutationCache#mutationcacheclear)

clear 方法可用于完全清除缓存并重新开始。

mutationCache.clear()
