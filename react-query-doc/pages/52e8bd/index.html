hydration

# hydration

[

# 脱水

](https://tanstack.com/query/latest/docs/react/reference/hydration#dehydrate)

dehydrate 创建缓存的冻结表示，稍后可以使用 Hydrate、useHydrate 或 hydrate 对其进行水合。这对于将预取查询从服务器传递到客户端或将查询持久保存到 localStorage 或其他持久位置很有用。默认情况下它只包括当前成功的查询。

import { dehydrate } from '@tanstack/react-query'
const dehydratedState = dehydrate(queryClient, {  shouldDehydrateQuery,})

选项

  • 客户端：查询客户端
    • 必需的
    • 应该脱水的queryClient
  • 选项：脱水选项
    • 选修的
    • 脱水突变：布尔值
      • 选修的
      • 是否脱水突变。
    • 脱水查询：布尔值
      • 选修的
      • 是否脱水查询。
    • shouldDehydrateMutation: (mutation: Mutation) => 布尔值
      • 选修的
      • 为缓存中的每个突变调用此函数
      • 返回 true 以将此突变包含在脱水中，否则返回 false
      • 默认版本仅包含暂停的突变
      • 如果您想在保留以前行为的同时扩展功能，请导入并执行 defaultShouldDehydrateMutation 作为 return 语句的一部分
    • shouldDehydrateQuery: (query: Query) => 布尔值
      • 选修的
      • 为缓存中的每个查询调用此函数
      • 返回 true 以将此查询包含在脱水中，否则返回 false
      • 默认版本只包括成功的查询，做 shouldDehydrateQuery: () => true 包括所有查询
      • 如果您想在保留以前行为的同时扩展功能，请导入并执行 defaultShouldDehydrateQuery 作为返回语句的一部分

Returns

  • 脱水状态：脱水状态
    • 这包括稍后补充 queryClient 所需的一切
    • 您不应该依赖此响应的确切格式，它不是公共 API 的一部分并且可以随时更改
    • 此结果不是序列化形式，如果需要，您需要自己做

[

# 限制

](https://tanstack.com/query/latest/docs/react/reference/hydration#limitations)

一些存储系统（例如浏览器 Web Storage API）要求值是 JSON 可序列化的。如果您需要对不能自动序列化为 JSON 的值进行脱水处理（例如 Error 或 undefined），您必须自己序列化它们。由于默认只包含成功的查询，要包含错误，您必须提供 shouldDehydrateQuery，例如：

// serverconst state = dehydrate(client, { shouldDehydrateQuery: () => true }) // to also include Errorsconst serializedState = mySerialize(state) // transform Error instances to objects
// clientconst state = myDeserialize(serializedState) // transform objects back to Error instanceshydrate(client, state)

[

# 水合物

](https://tanstack.com/query/latest/docs/react/reference/hydration#hydrate)

hydrate 将先前脱水的状态添加到缓存中。

import { hydrate } from '@tanstack/react-query'
hydrate(queryClient, dehydratedState, options)

选项

  • 客户端：查询客户端
    • 必需的
    • queryClient 将状态混合到
  • 脱水状态：脱水状态
    • 必需的
    • 水合物进入客户端的状态
  • 选项：HydrateOptions
    • 选修的
    • 默认选项：默认选项
      • 选修的
      • mutations: MutationOptions 用于水合突变的默认突变选项。
      • 查询：QueryOptions 用于混合查询的默认查询选项。
    • 上下文？：React.Context<QueryClient |未定义>
      • 使用它来使用自定义 React 查询上下文。否则，将使用 defaultContext。

[

# 限制

](https://tanstack.com/query/latest/docs/react/reference/hydration#limitations-1)

如果 dehydration 中包含的查询已经存在于 queryCache 中，hydrate 不会覆盖它们，它们将被静默丢弃。

[

# 使用水合物

](https://tanstack.com/query/latest/docs/react/reference/hydration#usehydrate)

useHydrate 将先前的脱水状态添加到 queryClient 中，该状态将由 useQueryClient() 返回。如果客户端已经包含数据，新查询将根据更新时间戳智能合并。

import { useHydrate } from '@tanstack/react-query'
useHydrate(dehydratedState, options)

选项

  • 脱水状态：脱水状态
    • 必需的
    • 补水的状态
  • 选项：HydrateOptions
    • 选修的
    • 默认选项：查询选项
      • 用于混合查询的默认查询选项。
    • 上下文？：React.Context<QueryClient |未定义>
      • 使用它来使用自定义 React 查询上下文。否则，将使用 defaultContext。

[

# 补水

](https://tanstack.com/query/latest/docs/react/reference/hydration#hydrate-1)

Hydrate 将 useHydrate 包装到组件中。当您需要类组件中的 hydrate 或在 QueryClientProvider 呈现的同一级别上需要 hydrate 时可能很有用。

import { Hydrate } from '@tanstack/react-query'
function App() {  return <Hydrate state={dehydratedState}>...</Hydrate>}

选项

  • 状态：脱水状态
    • 补水的状态
  • 选项：HydrateOptions
    • 选修的
    • 默认选项：查询选项
      • 用于混合查询的默认查询选项。
    • 上下文？：React.Context<QueryClient |未定义>
      • 使用它来使用自定义 React 查询上下文。否则，将使用 defaultContext。
