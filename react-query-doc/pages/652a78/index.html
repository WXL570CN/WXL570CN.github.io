QueryCache

# QueryCache

QueryCache 是 TanStack Query 的存储机制。它存储它包含的所有数据、元信息和查询状态。

通常，您不会直接与 QueryCache 交互，而是将 QueryClient 用于特定缓存。

import { QueryCache } from '@tanstack/react-query'
const queryCache = new QueryCache({  onError: error => {    console.log(error)  },  onSuccess: data => {    console.log(data)  },  onSettled: (data, error) => {    console.log(data, error)  },})
const query = queryCache.find({ queryKey: ['posts'] })

它可用的方法有：

  • 寻找 (opens new window)
  • 找到所有 (opens new window)
  • 订阅 (opens new window)
  • 清除 (opens new window)

选项

  • onError?: (error: unknown, query: Query) => 无效
    • 选修的
    • 如果某些查询遇到错误，将调用此函数。
  • onSuccess?: (data: unknown, query: Query) => 无效
    • 选修的
    • 如果某些查询成功，将调用此函数。
  • onSettled?: (data: unknown | undefined, error: unknown | null, query: Query) => 无效
    • 选修的
    • 如果某些查询已解决（成功或错误），将调用此函数。

[

# 全局回调

](https://tanstack.com/query/latest/docs/react/reference/QueryCache#global-callbacks)

QueryCache 上的 onError、onSuccess 和 onSettled 回调可用于在全局级别处理这些事件。它们与提供给 QueryClient 的 defaultOptions 不同，因为：

  • defaultOptions 可以被每个 Query 覆盖——全局回调将始终被调用。
  • defaultOptions 回调将为每个 Observer 调用一次，而全局回调只会为每个 Query 调用一次。

[

# 查询缓存.find

](https://tanstack.com/query/latest/docs/react/reference/QueryCache#querycachefind)

find 是一种稍微高级的同步方法，可用于从缓存中获取现有的查询实例。该实例不仅包含查询的所有状态，还包含查询的所有实例和底层内容。如果查询不存在，将返回 undefined。

注意：大多数应用程序通常不需要这，但在极少数情况下需要有关查询的更多信息时可以派上用场（例如，查看 query.state.dataUpdatedAt 时间戳以确定查询是否足够新鲜以用作初始值）

const query = queryCache.find({ queryKey })

选项

  • filters?: QueryFilters: 查询过滤器

Returns

  • 询问
    • 来自缓存的查询实例

[

# 查询缓存.findAll

](https://tanstack.com/query/latest/docs/react/reference/QueryCache#querycachefindall)

findAll 是更高级的同步方法，可用于从缓存中获取与查询键部分匹配的现有查询实例。如果查询不存在，将返回空数组。

注意：大多数应用程序通常不需要这，但在极少数情况下需要有关查询的更多信息时可以派上用场

const queries = queryCache.findAll({ queryKey })

选项

  • filters?: QueryFilters: 查询过滤器

Returns

  • 询问[]
    • 从缓存中查询实例

[

# 查询缓存.订阅

](https://tanstack.com/query/latest/docs/react/reference/QueryCache#querycachesubscribe)

subscribe 方法可用于订阅整个查询缓存，并获知缓存的安全/已知更新，例如查询状态更改或查询被更新、添加或删除

const callback = event => {  console.log(event.type, event.query)}
const unsubscribe = queryCache.subscribe(callback)

选项

  • 回调：（事件：QueryCacheNotifyEvent）=>无效
    • 每当查询缓存通过其跟踪更新机制（例如，query.setState、queryClient.removeQueries 等）进行更新时，都会调用此函数。不鼓励对缓存进行超出范围的突变，并且不会触发订阅回调

Returns

  • 退订：功能=>无效
    • 此函数将从查询缓存中取消订阅回调。

[

# 查询缓存.clear

](https://tanstack.com/query/latest/docs/react/reference/QueryCache#querycacheclear)

clear 方法可用于完全清除缓存并重新开始。

queryCache.clear()
