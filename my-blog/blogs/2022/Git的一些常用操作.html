    # Git相关

    # 一、commit规范

    • feat：添加新特性
    • refactor：代码重构，没有添加新功能或者修复
    • fix：修复bug
    • style：仅仅修改了空格、格式缩进、逗号等等，不改变代码逻辑
    • test：增加测试用例
    • chore：改变构建流程、或者增加依赖库、工具等
    • revert：回滚到上一个版本
    • docs：仅仅修改了文档

    # 二、修改已push至远程仓库的日志

    # 1、查看日志

    # 3、修改提交记录

    • 输入i进入编辑模式
    • 找到需要修改注释的一行，并将其头部的pick修改为edit
    • ESC退出编辑模式
    • 输入:wq!保存退出

    # 4、更正commit注释内容

    • 输入i进入编辑模式
    • 第一行即要修改的提交记录，将其修改为正确的注释
    • ESC退出编辑模式
    • 输入:wq!保存退出

    # 5、提交修改确认

    # 确认
git rebase --continue
# 提交
git push --force

    # 三、warning: LF will be replaced by CRLF in

    存在符号转义问题
    windows中的换行符为 CRLF， 而在linux下的换行符为LF，所以在执行git add时出现提示，解决办法：

    git config --global core.autocrlf false
