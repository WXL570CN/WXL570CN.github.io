    Git的一些常用操作

    WXL570CN 2022/6/4 Git

    # 一、commit规范

    • feat：添加新特性
    • refactor：代码重构，没有添加新功能或者修复
    • fix：修复bug
    • style：仅仅修改了空格、格式缩进、逗号等等，不改变代码逻辑
    • test：增加测试用例
    • chore：改变构建流程、或者增加依赖库、工具等
    • revert：回滚到上一个版本
    • docs：仅仅修改了文档

    # 二、修改已push至远程仓库的日志

    # 1、查看日志

    git log --oneline
    1

    # 2、选择要修改的提交记录

    # 修改版本号为0fefa3b的提交记录
git rebase -i HEAD~0fefa3b
    1
    2

    # 3、修改提交记录

    • 输入i进入编辑模式
    • 找到需要修改注释的一行，并将其头部的pick修改为edit
    • ESC退出编辑模式
    • 输入:wq!保存退出

    # 4、更正commit注释内容

    • 输入i进入编辑模式
    • 第一行即要修改的提交记录，将其修改为正确的注释
    • ESC退出编辑模式
    • 输入:wq!保存退出

    # 5、提交修改确认

    # 确认
git rebase --continue
# 提交
git push --force
